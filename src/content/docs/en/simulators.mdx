## Disambiguations

Below is a list of key terms with their specific meanings to avoid confusion in this context.

- [Physics engine - Wikipedia](https://en.wikipedia.org/wiki/Physics_engine) *"is computer software that provides an approximate simulation of certain physical systems, typically classical dynamics, including rigid body dynamics (including collision detection), soft body dynamics, and fluid dynamics."*.
- [Render engine (or Renderer)](https://en.wikipedia.org/wiki/Rendering_(computer_graphics)): is the piece of software that is in charge of generating "[...] a photorealistic or non-photorealistic image from input data such as 3D models" . Here you can find a [List of 3D rendering software - Wikipedia](https://en.wikipedia.org/wiki/List_of_3D_rendering_software).
- [Game engine - Wikipedia](https://en.wikipedia.org/wiki/Game_engine) : *"is a software framework primarily designed for the development of video games which generally includes relevant libraries and support programs such as a level editor, physics engine, renderer, assets management and more."*. Some example from a [List of game engines](https://en.wikipedia.org/wiki/List_of_game_engines).
- [Robotics simulator - Wikipedia](https://en.wikipedia.org/wiki/Robotics_simulator): *"is a simulator used to create an application for a physical robot without depending on the physical machine, thus saving cost and time."*
- Simulator Framework: a collection of multiples software components adjusted to work togetter....

@collinsReviewPhysicsSimulators2021 defines a simulator as: "an end-user software application that includes at least the following functionality:"
- physics engine
- collision detection and friction models (harley's note: is not part of the physics engine?)
- Graphical User Interface
- Import capabilities (mesh and scenes)
- API
- models for an array of joints, actuators and sensors readily available.

What makes a robotics simulator a robotics simulator:
- robot model support: Support for standard robotic models (URDF, SDF)
- sensors: LIDAR, cameras, IMU, force-torque
- Multi-body Dynamics (joint types): Simulates rigid and soft body interactions relevant to robots.
- Sensor Noise & Distortions: Allows realistic sensor imperfections for real-world applicability.
- physics Fidelity: More focus on realistic kinematics and dynamics rather than high-speed rendering.
- Direct API: for multiples programming languages
- communication APIs

- dm: discrete mechanics
- ik: inverse kinematics solvers (please find IK specialized packages in this list)
- id: inverse dynamics
- slam: simultaneous localization and mapping
- trj-opt: trajectory optimization
- plan: motion planning algorithms
- cv: computer vision
- urdf: urdf parser
- sdf: sdf parser
