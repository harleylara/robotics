## Prims

<Definition term="Prim">
    A Prim is the primary container object in USD: prims can contain (and order) other prims, creating a “namespace hierarchy” on a **[Stage](#stage)**, and prims can also contain (and order) properties that hold meaningful data.

Source: [USD - Glossary](https://openusd.org/release/glossary.html#id417)
</Definition>


## Layer

<Definition term="Layer">
A Layer is the atomic persistent container of scene description for USD.

Source: [USD - Glossary](https://openusd.org/release/glossary.html#usdglossary-layer)
</Definition>

- `.usda`: ASCII Text, human-readable format
- `.usdc`: USD Crate file format. High performance binary not human-readable format
- `.usd`: either Crate or Text
- `.usdz`: uncompressed and packaged format (.zip)

## Stage

<Definition term="Stage">
A stage is the USD abstraction for a scenegraph derived from a root USD file.

Source: [USD - Glossary](https://openusd.org/release/glossary.html#usdglossary-stage)
</Definition>
